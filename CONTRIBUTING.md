# Contributing to Agent Trust Protocol

Thank you for your interest in contributing to the Agent Trust Protocol (ATP)! This document provides guidelines and instructions for contributing to the project.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct. Please read it before contributing.

## How to Contribute

There are many ways to contribute to ATP:

1. **Report bugs**: If you find a bug, please create an issue with detailed information about how to reproduce it.
2. **Suggest features**: If you have an idea for a new feature, create an issue to discuss it.
3. **Improve documentation**: Help us improve our documentation by fixing errors or adding missing information.
4. **Submit code changes**: Implement new features or fix bugs by submitting pull requests.

## Development Setup

1. **Fork the repository**:
   ```bash
   git clone https://github.com/agent-trust-protocol/atp.git
   cd atp
   ```

2. **Install dependencies**:
   ```bash
   npm install
   ```

3. **Build the project**:
   ```bash
   npm run build
   ```

4. **Run tests**:
   ```bash
   npm run test
   ```

## Pull Request Process

1. **Create a branch**: Create a new branch for your changes.
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes**: Implement your changes, following our coding standards.

3. **Write tests**: Add tests for your changes to ensure they work correctly.

4. **Update documentation**: Update any relevant documentation.

5. **Submit a pull request**: Push your changes to your fork and submit a pull request to the main repository.

6. **Code review**: Wait for a maintainer to review your changes. You may need to make additional changes based on feedback.

## Coding Standards

- Follow the TypeScript coding style used in the project
- Write comprehensive tests for new functionality
- Document your code with JSDoc comments
- Follow W3C standards for DID and VC implementations
- Maintain backward compatibility in public APIs

## Commit Message Guidelines

We follow conventional commits for our commit messages:

- `feat`: A new feature
- `fix`: A bug fix
- `docs`: Documentation changes
- `style`: Code style changes (formatting, etc.)
- `refactor`: Code changes that neither fix bugs nor add features
- `test`: Adding or updating tests
- `chore`: Changes to the build process or auxiliary tools

Example: `feat(identity): add key rotation endpoint`

## License

By contributing to ATP, you agree that your contributions will be licensed under the project's [Apache 2.0 License](LICENSE).

## Questions?

If you have any questions, feel free to create an issue or reach out to the maintainers.
